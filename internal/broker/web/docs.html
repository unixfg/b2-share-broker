Upload

Uploads accept one multipart file and return a queued processing job plus the future share URL.

export TOKEN="eyJ..."
export FILE="./clip.mp4"

curl -sS -X POST "https://share.doesthings.online/api/uploads" \
  -H "Authorization: Bearer ${TOKEN}" \
  -F "file=@${FILE}"

Status

Poll the job until it is completed, failed, or canceled.

curl -sS "https://share.doesthings.online/api/uploads/${JOB_ID}" \
  -H "Authorization: Bearer ${TOKEN}"

History

History is owner-only. Search matches filename, slug, content type, and status.

curl -sS "https://share.doesthings.online/api/shares?q=mp4" \
  -H "Authorization: Bearer ${TOKEN}"

Delete

Deleting a share removes the public alias and deletes the B2 object only when no visible aliases still reference it.

curl -sS -X DELETE "https://share.doesthings.online/api/shares/${SLUG}" \
  -H "Authorization: Bearer ${TOKEN}"

Notes

  • Uploads are limited to one file at a time.
  • The current upload limit is 2 GiB.
  • Public share URLs under /s/ are unlisted but readable by anyone with the URL.
  • Ready shares return a 302 redirect to the native public B2 object URL.
  • Public media CORS is a manual read-only allowlist on both the share redirect and B2 bucket.
